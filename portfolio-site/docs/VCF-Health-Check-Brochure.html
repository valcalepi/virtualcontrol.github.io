Your VCF Environment Deserves Better Than "It Seems Fine"

The Problem You Already Know About

You are running VMware Cloud Foundation. Your environment has vCenter, SDDC Manager, NSX, ESXi hosts, certificates, DNS records, datastores, vSAN clusters, and dozens of interconnected services — all of which need to be healthy for your infrastructure to function.

Right now, how do you know everything is healthy?

You check vCenter manually. You SSH into SDDC Manager. You open NSX and look around. You hope nothing slipped through the cracks. When something does break, you spend hours figuring out what went wrong, tracing it back through logs, and explaining to stakeholders why it happened.

A certificate expired silently. DNS stopped resolving for a management component. A vSAN disk group went degraded and nobody noticed. An SDDC Manager backup has not run in 47 days. A vCenter service crashed and is not running. ESXi hosts drifted out of compliance.

These are not hypothetical problems. They are the problems that wake you up at 2 AM, cost your company thousands in downtime, and erode trust with the clients you serve.

What Gets Checked

VCF Health Check is not a generic monitoring tool. It was built specifically for VMware Cloud Foundation environments and understands the relationships between components that matter.

Infrastructure and Network (14 Checks)

DNS forward and reverse resolution for every management component. SSL certificate expiry monitoring with advance warnings. Network latency tracking to vCenter, SDDC Manager, and NSX. ESXi host health checks running in parallel for speed.

If a DNS record is wrong, you will know. If a certificate is expiring in 30 days, you will know. If latency to a management endpoint is spiking, you will know.

vCenter Server (33 Checks)

The deepest coverage of any VCF health tool. API reachability, version and build detection, system health (CPU, memory, storage, swap, database), VM inventory, ESXi host connection states, cluster HA and DRS configuration, datastore capacity per-datastore, vSAN cluster and disk group health, vSAN resync and rebuild status, NTP time sync, root password expiry, individual service status with deep checks (vpxd, stsd, vpostgres, rhttpproxy), active alarms, VM snapshot age, license expiry, VCSA disk partition utilization, storage policies, content library sync, VCF BOM compliance, and 24-hour event log errors.

33 checks. Every one of them matters. Every one of them has caused a real outage somewhere.

SDDC Manager (16 Checks)

API reachability with retry and backoff, version detection, NSX and vCenter status from SDDC Manager's perspective, management component health, certificate inventory with 60-day warnings, backup age and status, system prechecks, credential and password expiry, in-progress task detection, resource lock detection, host drift detection, depot connectivity, stale task identification, and lifecycle update availability.

SDDC Manager is the control plane of your entire VCF deployment. If it is unhealthy, everything downstream is at risk.

NSX Manager (11 Checks)

API reachability with VIP and individual node fallback, control cluster status, management cluster status, transport node state, open alarms with severity classification, admin password expiry, version detection, edge cluster member health, distributed firewall rule count thresholds, certificate validation, and transport zone membership.

NSX problems cascade fast. A failing transport node or a degraded control cluster can take down your entire network overlay in minutes.

VCF Operations and ARIA Suite (4 Checks)

Suite API reachability, node online status, collector health, and adapter data receiving verification.

Fleet and vRSLCM (2 Checks)

API reachability and environment deployment status.

The Report That Changes How You Work

After every health check, VCF Health Check generates a comprehensive report. Not a wall of text. Not a raw log dump. A professionally designed, interactive report that you can hand to your CTO, your client, or your operations team.

  • Executive view (high-level summary for leadership) and Technical view (full details for engineers)

  • Diff view to compare two runs side by side and see exactly what changed
  • Dependency map showing component relationships and health status
  • Score trend chart showing your health trajectory over time
  • 90-day heatmap calendar — a color-coded grid showing your daily health grade for the past three months
  • SLA uptime tracker with per-component uptime percentages
  • Network latency table with color-coded performance badges
  • Remediation playbook export — a ready-to-use checklist of exactly what to fix and how
  • Keyboard shortcuts for power users
  • Mobile responsive for checking on your phone

Automation That Runs While You Sleep

Set it and forget it. VCF Health Check generates scheduling configurations for:

  • Windows Task Scheduler (one command generates the XML)
  • Linux cron (one command generates the cron entry)
  • systemd timers (one command generates the timer files)

Run health checks every morning at 6 AM before your team arrives. Run them every hour during a maintenance window. Run them weekly and get a summary every Monday morning. Configure it once, and your VCF environments are monitored continuously without any manual effort.

Auto-Remediation for Safe Fixes

Some problems have obvious, safe fixes. Stale SDDC Manager tasks that have been stuck for days. Configuration items that drifted from expected values. VCF Health Check can automatically fix these issues during a run with the —fix flag, saving your team time and reducing the backlog of minor issues that pile up.

For everything else, the remediation playbook tells you exactly what to do, step by step, with estimated time to fix each issue.
